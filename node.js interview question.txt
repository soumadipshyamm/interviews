## ‚öôÔ∏è **1. Modules and `require` / `import-export`**

> **Interview Answer:**
> Node.js uses modules to organize code into reusable files.
> There are two main systems:

* **CommonJS** ‚Üí Uses `require()` to import and `module.exports` to export.
* **ES Modules (ESM)** ‚Üí Uses `import` and `export` keywords, and are modern JavaScript standard modules.

‚úÖ *Example:*

```js
// CommonJS
const fs = require('fs');

// ES Module
import fs from 'fs';
```

> **Key difference:** CommonJS is synchronous (older system), while ES Modules are asynchronous and follow modern JS syntax.

## üåç **2. Global Objects**

> **Interview Answer:**
> Global objects in Node.js are available everywhere without importing.
> Some common ones:

* `console` ‚Üí For logging
* `process` ‚Üí Info about the running Node process (like environment, PID)
* `__dirname` ‚Üí Directory path of the current file
* `__filename` ‚Üí Full path of the current file
* `global` ‚Üí Like `window` in browser, global namespace in Node

‚úÖ *Example:*

```js
console.log(__dirname); 
console.log(process.env.NODE_ENV);
```

---

## üìÅ **3. File System Module (`fs`)**

> **Interview Answer:**
> The `fs` module is used to interact with the file system ‚Äî reading, writing, updating, or deleting files.
> It supports both **synchronous** and **asynchronous** methods.

‚úÖ *Example:*

```js
const fs = require('fs');
fs.readFile('file.txt', 'utf8', (err, data) => console.log(data));
```

> **Tip:** Prefer asynchronous methods to avoid blocking the event loop.

---

## üß≠ **4. Path Module (`path`)**

> **Interview Answer:**
> The `path` module helps work with file and directory paths in a platform-independent way.

‚úÖ *Example:*

```js
const path = require('path');
console.log(path.join(__dirname, 'folder', 'file.txt'));
```

> It handles separators automatically and provides helpers like `basename`, `dirname`, and `extname`.

---

## üíª **5. OS Module (`os`)**

> **Interview Answer:**
> The `os` module provides system-related information such as memory, CPU, and platform details.

‚úÖ *Example:*

```js
const os = require('os');
console.log(os.platform(), os.totalmem(), os.freemem());
```

> Commonly used to monitor system performance or display environment details.

---

## ‚ö° **6. Events Module (`EventEmitter`)**

> **Interview Answer:**
> Node.js is **event-driven**. The `events` module provides the `EventEmitter` class, which lets you create and listen to custom events.

‚úÖ *Example:*

```js
const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('greet', () => console.log('Hello!'));
emitter.emit('greet');
```

> Used heavily in Node‚Äôs core modules ‚Äî for example, HTTP servers, streams, and sockets.

## üåä **7. Streams**

> **Interview Answer:**
> Streams are used to process data piece by piece instead of loading it all at once (helps with large files or continuous data).

There are **4 types**:

1. **Readable** ‚Äì For reading data
2. **Writable** ‚Äì For writing data
3. **Duplex** ‚Äì Both readable and writable
4. **Transform** ‚Äì Modify data as it passes through

‚úÖ *Example:*

```js
const fs = require('fs');
fs.createReadStream('input.txt').pipe(fs.createWriteStream('output.txt'));
```

> **Advantage:** Streams improve performance and memory efficiency.
